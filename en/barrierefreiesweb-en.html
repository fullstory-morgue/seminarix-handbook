Barrier-free Web

Creation of universally accessible Internet-Sites

The term 'barrier-free web' is used for Internet-Sites which are created, so that they are readable by people with sight problems or colour blindness.

Internet users with simple browsers or with disabled JavaScript, without Flash or with disabled Graphics, that search the Internet for information, often realise that Web-masters have not catered for them.

Following some simple rules, you can help provide everyone with information, without leaving someone out.

  1. Graphical links and graphics which are important for understanding the site, are supplied with ALT-Tags. 
  2. A clean HTML-Code, in which all commands are closed using the appropriate Tag, makes the site more readable for Screen-Readers.
  3. Javascript, which produces a clean HTML-Code, creates no problems.
  4. Frames are also represented in the various access-solutions and so they can also be used. This also holds for frame navigation.
  5. Problematic, from the barrier-free web point of view, are embedded Java-Applets, that offer links which could be as easily written in HTML.
  6. Multimedia-Applications also cause problems. They should not contain any information which is essential to understanding the site.

Additional and more detailed information can be found in the website Web Accessibility Initiative (WAI). There you can find links for a lot of additional suggestions and references for the optimisation of the Internet performance for all popular browsers:

  1. Netscape, Firefox or Mozilla
  2. Internet Explorer
  3. Opera

As an educator, the content of your website's information could be of value to those with sight impairments.
