Seminarix is a Live-medium based on sidux, which is developed for the education of teachers

sidux-Seminarix usage information

The program starter

Each program can be started with a double click on the program's icon or by clicking on the start button. There are programs listed in the program starter which are not contained on the live-medium (the size of a live-medium is limited to approx. 700 MB respectively 2000MB depending on the medium). For those you can select "only not installed" from the drop-down menu which is situated in the right corner above the selection window. With an Internet connection it is possible to easily install programs which are listed in the menu system but are not contained on the live-medium. You recognize those programs because they have no "Start" button and no tick in the box that is situated next to the program icon. Clicking the box and then the "show changes" button you can install the chosen programs or cancel by clicking on the respective buttons.

Usually you can find, in each program's description, a link to the home page of the program in which you can find further information about the program.

Create a permanent home directory

With the help of the sidux live-medium you can create a permanent home directory. By doing that the general settings of the graphics card, keyboard, mouse or network will not be stored but only you personal settings and data. With that it is possible to use them again after restarting the live-medium.

With sidux-Seminarix you can setup a permanent (persistent) live-system on a writable partition (for example on a USB-stick or hard disk)(an ISO copy). For that the boot options 'fromiso' and 'persist' are used. The persistent data is stored in a subdirectory of the fromiso partition. The procedure is very well explained here.

To for example be able to work completely with out a live-medium, you are offered the possibility to create a boot-able sidux-USB-Stick with its own data storage and a self chosen programs list.

The use of the 'persist' boot option on a FAT file system (most USB-Sticks are pre-formated with FAT), requires a big file which has to be setup as a loop device. This file has to be formated (ext3 is recommended).

This file should be named sidux-rw and it should be placed in the root directory of the FAT partition (USB-Stick with FAT file system). If the partition is for example mounted and writable on /media/sdb1 and you want to create a 1GB file to use as your persistent drive you can use the following commands (no need for root if you can write to the file):

disc="/media/sdb1" size=1024 dd if=/dev/zero of=${disc}/sidux-rw bs=1M count=${size} && echo 'y' | /sbin/mkfs.ext3 ${disc}/sidux-rw

NTFS partitions, commonly used for Windows NT/2000/XP/Vista (TM) installations, CANNOT be used for persistence.
